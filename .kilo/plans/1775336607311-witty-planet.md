# Plan : Déployer le Frontend AirSentinel via Fork

## Objectif
Forker ce projet et déployer le frontend Next.js sur Vercel (recommandé) ou Render.

---

## Étapes pour le Fork

### 1. Forker sur GitHub
1. Aller sur https://github.com/fofack/AirSentinel
2. Cliquer "Fork" (coin haut droite)
3. Choisir son compte GitHub
4. Nommer le repo puis "Create fork"

### 2. Cloner le fork localement (optionnel)
```bash
git clone https://github.com/VOTRE_COMPTE/AirSentinel.git
cd AirSentinel
```

---

## Déployer sur Vercel (Recommandé pour Next.js)

### Via Vercel CLI
```bash
npm i -g vercel
cd pwa/airsentinel
vercel login
vercel --prod
```

### Via Dashboard Vercel
1. Aller sur https://vercel.com
2. "Add New Project" → "Import Git Repository"
3. Sélectionner le fork AirSentinel
4. Configurer :
   - Framework Preset: Next.js
   - Root Directory: `pwa/airsentinel`
   - Build Command: `npm run build` (ou laisser par défaut)
   - Output Directory: `.next` (ou laisser par défaut)
5. Variables d'environnement à ajouter :
   ```
   NEXT_PUBLIC_API_URL=https://votre-api.onrender.com
   ```
6. Deploy

---

## Variables d'environnement

### Frontend (pwa/airsentinel/)
Créer `.env.local` :
```env
NEXT_PUBLIC_API_URL=http://localhost:8000
# Ou en production:
NEXT_PUBLIC_API_URL=https://airsentinel-api.onrender.com
```

### À configurer sur Vercel
- `NEXT_PUBLIC_API_URL` = URL du backend déployé

---

## Ordre de déploiement recommandé

1. **Backend** : Déployer sur Render (voir plan précédent)
2. **Frontend** : Déployer sur Vercel avec `NEXT_PUBLIC_API_URL` pointant vers le backend
